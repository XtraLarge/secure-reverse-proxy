Sanitized deploy fixture for local and server-side tests.

Contents prepared by `tests/prepare-local-test-assets.sh`:

- `.env`: non-secret runtime settings with realistic defaults
- `sites-enabled/`: only configs matching `OIDC_COOKIE_DOMAIN`
- `AddOn/`: selected production-like snippets for the same domain
- `ssl/`: generated by copying the existing test certificate to the same domain

Default target outside the repo: `/data/oidcproxy-test`

The default fixture is intentionally single-domain. A running stack should not
mix unrelated domains with a different `OIDC_COOKIE_DOMAIN`, because
mod_auth_openidc expects the cookie domain to match the served hostnames.

This fixture must stay free of production credentials.

For `dc`-managed Docker hosts there are two helper installers:

- `tests/install-dc-test-compose.sh`: test stack with published ports
- `tests/install-dc-proxy-compose.sh`: proxy-style stack for `/data/proxy` with `VLan10` and optional `keycloak` profile
